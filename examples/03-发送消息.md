# 场景：给客户发送消息

## 用户可能会这样说
- "给张三发个消息"
- "通知所有 VIP 客户明天有活动"
- "提醒未付款的客户"
- "群发一个促销信息"

## AI 思考过程

1. **识别意图**：用户想发送消息
2. **确定接收者**：
   - 单个客户 → 需要 `chat_user_id`
   - 批量客户 → 先查询筛选，再批量发送
3. **确定消息内容**：从用户输入中提取或询问
4. **选择脚本**：`send-message.py`（需要确认脚本存在）
5. **确认发送**：批量发送前需要用户确认

## 调用命令

### 给单个客户发消息
```bash
python3 scripts/send-message.py --chat-user-id "0012c2a7..." --msg "您好，您的订单已发货"
```

### 给批量客户发消息（先查询）
```bash
# 1. 先查 VIP 客户
python3 scripts/query-customers.py --tags "VIP" --page-size 100

# 2. 提取客户 ID 列表，批量发送
python3 scripts/batch-send-message.py --chat-user-ids "ID1,ID2,ID3" --msg "活动通知..."
```

## 预期输出格式

```
✅ 消息发送成功
========================================
接收者：张三 (0012c2a7...)
消息内容：您好，您的订单已发货
发送时间：2026-03-17 19:30:00
状态：已送达
```

## ⚠️ 重要注意事项

### 批量发送前必须确认
```
AI: 您想给 156 个 VIP 客户发送活动通知，确认要发送吗？
用户：确认
AI: 好的，正在发送...
```

### 避免敏感词
- 不要发送营销骚扰内容
- 遵守 SaleSmartly 使用政策
- 建议添加退订选项

## 错误处理

| 错误情况 | AI 应该回复 |
|---------|-----------|
| 客户 ID 不存在 | "未找到该客户，请检查客户 ID 是否正确" |
| 消息内容为空 | "请告诉我您要发送什么内容" |
| 发送失败 | "消息发送失败，可能是客户已拉黑或账号异常" |
| 频率限制 | "发送太快了，请稍后再试（SaleSmartly 限制每分钟最多发送 10 条）" |
