# 技能包意图识别优化指南

**目标**: 确保 salesmartly-api-skills 在不同 LLM 模型下都有稳定的意图识别准确率

**适用对象**: 技能包开发者  
**创建时间**: 2026-03-24

---

## 🔍 问题背景

### 现状

```
技能包用户 → 使用各种 LLM 模型 → 意图识别准确率参差不齐
              ├─ Qwen3.5-plus (95%+)
              ├─ Qwen3.5-flash (85%)
              ├─ Qwen2.5-72B (80%)
              ├─ GPT-3.5-Turbo (75%)
              └─ 小模型 (<7B, 60%)
```

### 核心问题

1. **技能包无法控制用户用什么模型**
2. **不同模型对 SKILL.md 的理解能力差异大**
3. **弱模型容易误解意图、提取错误参数**

---

## 💡 解决方案：技能包自适应设计

### 架构思路

```
┌─────────────────────────────────────┐
│  SKILL.md (技能包说明文件)           │
│                                     │
│  ✅ 明确的触发词列表                 │
│  ✅ 标准化的意图映射表               │
│  ✅ Few-shot 示例（多模型兼容）      │
│  ✅ 降级策略指导                     │
│  ✅ 参数验证规则                     │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│  OpenClaw AI (各种模型)              │
│                                     │
│  L1 模型 → 完整理解 SKILL.md         │
│  L2 模型 → 依赖触发词 + 映射表       │
│  L3 模型 → 仅依赖触发词              │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│  脚本执行 (稳定性一致)               │
└─────────────────────────────────────┘
```

#### 1.4 降级策略指导

```markdown
## ⚠️ 降级策略（AI 必读！）

### 何时需要澄清？

**置信度低时，不要调用脚本，先问清楚：**

| 情况 | 示例 | 正确做法 |
|------|------|---------|
| 时间不明确 | "查客户" | 问："请问查哪个时间范围？" |
| 对象不明确 | "分配客户" | 问："请问分配给哪个客服？" |
| 动作不明确 | "客户张三" | 问："请问要查询、更新还是分配？" |
| 参数冲突 | "昨天和今天的客户" | 问："请问查昨天还是今天？" |

### 澄清话术模板

```
我没理解{不明确的部分}，请问您是想：
1. {选项 1}
2. {选项 2}
3. {选项 3}
```

### 置信度判断

**调用脚本前，自我评估：**

- ✅ 置信度≥0.8：直接调用
- ⚠️ 置信度 0.5-0.8：可以调用，但告知用户假设
- ❌ 置信度<0.5：先澄清，不要调用

**示例**:
```
⚠️ 我理解您想查询昨天的客户，对吗？
（如果不对，请说明具体时间范围）

✅ 确认无误，正在查询...
```
```

#### 1.5 参数验证规则

```markdown
## ✅ 参数验证（AI 必须检查！）

### 必填参数检查

| 脚本 | 必填参数 | 缺失时处理 |
|------|---------|-----------|
| `assign-session.py` | `--chat-user-id`, `--member-id` | 问："请问分配给哪个客服？" |
| `query-messages.py` | `--chat-user-id` 或 `--session-id` | 问："请问查哪个客户的聊天记录？" |
| `update-customer.py` | `--chat-user-id` | 问："请问更新哪个客户？" |

### 参数格式验证

| 参数 | 正确格式 | 错误格式 | 验证规则 |
|------|---------|---------|---------|
| `--days` | `7` (整数) | "七天" | 必须是数字 |
| `--date` | `2026-03-24` | "昨天" | YYYY-MM-DD 格式 |
| `--phone` | `8613800138000` | `13800138000` | 加区号，不含 + |
| `--chat-user-ids` | `id1,id2,id3` | `["id1","id2"]` | 逗号分隔字符串 |

### 参数边界检查

| 参数 | 最小值 | 最大值 | 超出处理 |
|------|-------|-------|---------|
| `--page-size` | 1 | 100 | 自动修正为 100 |
| `--days` | 1 | 365 | 问："时间范围太大，确定要查 365 天吗？" |
| `--chat-user-ids` | 1 个 | 100 个 | 超过 100 个时分批处理 |
```

#### 2.2 自动化测试脚本

创建 `tests/test-intent.py`:

```python
#!/usr/bin/env python3
"""
意图识别测试脚本

测试不同模型下，SKILL.md 的意图识别准确率
"""

import json
import sys
from pathlib import Path

# 测试用例
TEST_CASES = [
    {
        "input": "昨天的客户",
        "expected": {
            "script": "query-customers.py",
            "params": {"days": 1, "filter-by": "created"}
        }
    },
    {
        "input": "查张三的聊天记录",
        "expected": {
            "script": "query-messages.py",
            "params": {"chat-user-id": "张三"}
        }
    },
    # ... 更多用例
]

def test_intent(model_name: str):
    """测试指定模型的意图识别"""
    print(f"\n测试模型：{model_name}")
    print("=" * 60)
    
    passed = 0
    failed = 0
    
    for case in TEST_CASES:
        # 调用 LLM（模拟 OpenClaw 行为）
        result = call_llm(model_name, case["input"])
        
        # 验证结果
        if result["script"] == case["expected"]["script"]:
            if result["params"] == case["expected"]["params"]:
                print(f"✅ {case['input']}")
                passed += 1
            else:
                print(f"⚠️  {case['input']} - 参数错误")
                failed += 1
        else:
            print(f"❌ {case['input']} - 脚本错误")
            failed += 1
    
    print(f"\n结果：{passed} 通过，{failed} 失败")
    return passed, failed

if __name__ == "__main__":
    models = sys.argv[1:] or ["qwen3.5-plus", "qwen3.5-flash", "default"]
    
    for model in models:
        test_intent(model)
```

你是 SaleSmartly 助手，专门帮助用户管理客户、会话和销售数据。

## 可用工具

你只能使用以下脚本（不要自己发明参数）：

| 脚本 | 功能 | 常用参数 |
|------|------|---------|
| query-customers.py | 查询客户列表 | --days, --filter-by, --tags |
| query-messages.py | 查询聊天记录 | --chat-user-id, --days |

   - "昨天" → --days 1
   - "今天" → --today
   - "最近 7 天" → --days 7
   - "上周" → --start-date YYYY-MM-DD --end-date YYYY-MM-DD

2. **对象关键词** → 提取 ID
   - "客户张三" → --chat-user-id 张三

3. **动作关键词** → 选择脚本
   - "查/查询/看看" → query-*.py
   - "发/发送" → send-message.py
   - "分配/给" → assign-session.py

必须输出 JSON：
```json
{
  "script": "脚本名.py",
  "params": {"参数名": "参数值"},

  "reasoning": "识别理由"
}
```

- 0.9-1.0: 非常明确，直接调用
- 0.7-0.9: 基本明确，可以调用
- 0.5-0.7: 不太明确，建议澄清
- <0.5: 非常模糊，必须澄清
