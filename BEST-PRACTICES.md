## 🚀 快速上手（5 分钟开始使用）

### 第一步：安装 OpenClaw（如果你还没有）

```bash
# 安装 OpenClaw
npm install -g openclaw

# 启动服务
openclaw gateway start
```

### 第二步：安装 SaleSmartly Skills

在 OpenClaw 对话中说：

```
安装 https://github.com/Sale-Smartly/salesmartly-api-skills 技能
```

### 第三步：配置 API Key

```bash
cd ~/.openclaw/workspace/skills/salesmartly-api-skills
cp api-key.json.example api-key.json
# 编辑 api-key.json，填入你的 API Key 和 Project ID
```

**获取 API Key**：SaleSmartly 后台 → 设置 → API 集成 → 创建 API Key

### 第四步：开始对话

现在你可以像这样和 AI 对话：

```
"昨天新增了多少客户？"
"帮我查一下会话 j894hri 的聊天记录"
"推送今天的销售报告到钉钉"
"哪个客服的会话量最高？"
"检查有没有负面反馈"
```

## 📊 典型使用场景

### 场景 1：每日销售复盘

**时间**：每天上午 9:00

**传统做法**：
1. 登录 SaleSmartly 后台
2. 进入报表页面
3. 选择昨天日期
4. 导出数据
5. 手动整理发送给团队

**AI Skills 做法**：
```
"推送昨天的销售报告到钉钉"
```

**结果**：AI 自动生成报告并推送到钉钉群，包含：
- 新增客户数
- 成交订单数
- 会话转化率
- 客服绩效排名

---

### 场景 2：客户反馈实时监控

**痛点**：客户投诉发现太晚，已经造成损失

**AI Skills 解决方案**：

配置敏感词监控（在 `api-key.json` 中）：
```json
{
  "feedback_keywords": ["投诉", "退款", "差评", "垃圾", "骗子"],
  "dingtalk_webhook": "https://oapi.dingtalk.com/robot/send?access_token=xxx"
}
```

当客户消息包含敏感词时，AI 立即推送告警：
```
⚠️ 负面反馈告警
客户：张三 (ID: 12345)
会话：abc123
敏感词：投诉
时间：2026-03-20 10:35
消息内容："我要投诉你们的产品..."
```

**价值**：5 分钟内响应，避免事态升级。

---

### 场景 3：WhatsApp 设备批量管理

**场景**：你有 10+ 个 WhatsApp 设备需要管理

**传统做法**：逐个登录检查状态，手动扫码

**AI Skills 做法**：
```
"查看所有 WhatsApp 设备状态"
```

**输出**：
```
设备总数：12
在线：8
离线：2
扫码中：2

离线设备列表：
- 设备 A (138****1234) - 离线 2 小时
- 设备 B (139****5678) - 离线 30 分钟
```

然后：
```
"获取设备 A 的二维码"
```

AI 返回二维码图片，扫码即可重新连接。

### 场景 4：展会客户批量跟进

**场景**：参加完展会，需要跟进 100+ 潜在客户

**AI Skills 做法**：

1. 导入客户列表：
```
"从 Excel 导入 100 个展会客户，打上标签'2026 广州展会'"
```

2. 批量分析：
```
"分析展会客户的来源分布"
```

3. 制定跟进计划：
```
"列出 3 天内未回复的展会客户"
```

**结果**：原本需要 2 天的工作，现在 30 分钟完成。

## 🛠️ 高级技巧

### 技巧 1：组合查询

```
"查询最近 7 天新增客户，并统计来源分布"
```

AI 会自动组合多个脚本完成复杂查询。

### 技巧 2：定时任务

配置 Cron 自动任务：
```
"每天早上 9 点推送销售报告到钉钉"
```

AI 会创建定时任务，无需手动执行。

### 技巧 3：自定义报表

```
"创建一个报表，包含客户数、会话数、转化率，每周一推送"
```

AI 会根据你的需求生成定制化报表脚本。

### 技巧 4：多项目切换

如果你有多个 SaleSmartly 项目：
```bash
# 创建多个配置文件
cp api-key.json api-key-project1.json
cp api-key.json api-key-project2.json

# 使用时指定配置
python scripts/query-customers.py --config api-key-project1.json
```

## ❓ 常见问题

### Q: 需要编程基础吗？
**A**: 不需要。所有操作通过自然语言对话完成，AI 会自动调用对应的脚本。

### Q: 数据安全吗？
**A**: 所有脚本本地执行，API Key 存储在本地配置文件，不上传任何数据到外部服务器（除了 SaleSmartly 官方 API）。

### Q: 可以自定义推送渠道吗？
**A**: 目前支持钉钉，企业微信、飞书、Slack 正在开发中（v1.2 版本）。

### Q: 多个客服可以共用吗？
**A**: 可以。支持多配置文件切换，每个客服使用独立的 API Key。

### Q: 遇到问题怎么办？
**A**: 
1. 查看 `scripts/README.md` 完整文档
2. 在 OpenClaw 中问 AI："XXX 功能怎么用"
3. 提交 GitHub Issue
4. 钉钉群求助
